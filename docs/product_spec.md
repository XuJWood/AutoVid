> 版本：v2.1
> 更新日期：2026-04-06
> 定位：AI短剧创作平台
> 目标用户：AI视频创作者/供应商

---

## 一、产品概述

### 1.1 产品定位

AutoVid 是面向 **AI视频创作者和内容供应商** 的一站式AI视频生产平台，提供两大核心功能：

| 入口 | 功能 | 适用场景 |
|-----|------|---------|
| **短剧创作** | 文字→剧本→角色→分镜→完整短剧 | 连续剧、系列内容 |
| **视频生成** | 人物形象→一致性视频 | 单条视频、角色IP打造 |

### 1.2 核心差异化

| 维度 | 传统工具 | AutoVid |
|-----|---------|---------|
| 模型配置 | 固定模型 | 可自由切换多种模型 |
| 提示词控制 | 黑盒生成 | 可查看和修改提示词 |
| 角色一致性 | 每镜随机 | Character ID锁定 |
| 双模式 | 单一功能 | 短剧+单视频双入口 |

## 三、模型配置系统

### 3.1 模型配置入口

**位置**：系统设置 → 模型配置

**配置维度**：

| 配置项 | 用途 | 可选模型 |
|-------|------|---------|
| **剧本生成模型** | 生成剧本、角色设定、分镜脚本 | GPT-4、Claude、通义千问、文心一言、DeepSeek |
| **图像生成模型** | 生成角色形象、分镜图 | MidJourney、DALL-E 3、Stable Diffusion、即梦AI |
| **视频生成模型** | 图生视频 | 可灵AI、Seedance、Runway、Pika、Sora |
| **语音合成模型** | 配音 | Azure TTS、Edge TTS、阿里云TTS、ElevenLabs |

```json

      "params": {
        "temperature": 0.7,
        "max_tokens": 4000
      }
    },

    "image_generation": {
      "provider": "midjourney",
      "api_key": "${MIDJOURNEY_API_KEY}",
      "params": {
        "style": "raw",
        "quality": "high",
        "aspect_ratio": "1:1"
      }
    },

    "video_generation": {
      "provider": "kling",
      "api_key": "${KLING_API_KEY}",
      "params": {
        "model": "v1",
        "duration": 5,
        "fps": 24,
```

## 四、提示词修改系统

### 4.1 提示词修改入口

**可在以下环节修改提示词**：

| 环节 | 触发时机 | 可修改内容 |
|-----|---------|-----------|
| 剧本生成 | 点击"生成剧本"前 | 剧本生成提示词模板 |
| 角色生成 | 点击"生成角色"前 | 角色形象描述提示词 |
| 分镜生成 | 点击"生成分镜"前 | 分镜画面描述提示词 |
| 视频生成 | 点击"生成视频"前 | 视频动作描述提示词 |

### 4.2 提示词拼接逻辑

```
系统默认提示词模板
        +
用户自定义补充提示词
        =
最终发送给模型的提示词
```

**示例**：

```
【系统默认模板】
你是一位专业的短剧编剧。请根据以下要求生成短剧剧本：
- 题材：{genre}
- 时长：{duration}秒
- 角色数：{character_count}
- 基调：{tone}

请生成包含角色设定、场景描述、分镜脚本的结构化剧本。

【用户补充提示词】（可选）
{user_prompt}

【用户输入】
{user_input}
```

### 4.4 各环节提示词模板

#### 4.4.1 剧本生成提示词

```markdown
# 系统默认模板

你是一位专业的短剧编剧。请根据用户输入生成完整的短剧剧本。

## 输出要求

1. **角色设定**
   - 每个角色的姓名、年龄、职业
   - 外貌特征描述（用于后续生成形象）
   - 性格特点、说话风格

2. **场景列表**
   - 场景名称、环境描述
   - 时间设定（白天/夜晚）
   - 氛围基调

3. **分镜脚本**
   - 每个镜头的：镜头类型、画面描述、角色动作、对白台词
   - 时长预估
   - 情绪标注

## 格式要求
请以JSON格式输出，结构如下：
{
  "title": "剧名",
  "characters": [...],
  "scenes": [...]
}

# 用户补充
{user_prompt}

# 用户输入
{user_input}
```

```markdown
- 外貌：{character_appearance}
- 服装：{character_clothing}

- 生成{style_count}种不同风格：写实、动漫、电影质感、水墨风
- 每种风格生成{count_per_style}张
- 确保面部特征清晰，适合作为角色参考图
- 生成三视图：正面、侧面、背面

## 输出格式
高质量人物肖像，{aspect_ratio}，{quality}
```

## 八、API接口设计

### 8.1 模型配置接口

```yaml
# 获取模型配置
GET /api/v1/model-config

# 更新模型配置
PUT /api/v1/model-config

# 测试模型连接
POST /api/v1/model-config/test
{
  "provider": "openai",
  "api_key": "sk-xxx"
}
```

### 8.2 提示词接口

```yaml
# 获取提示词模板
GET /api/v1/prompt-templates

# 创建提示词模板
POST /api/v1/prompt-templates
{
  "name": "甜宠剧专用",
  "type": "script",
  "template": "..."
}

# 更新提示词模板
PUT /api/v1/prompt-templates/{id}
```

### 8.3 生成接口

```yaml
# 生成剧本（带提示词覆盖）
POST /api/v1/projects/{id}/script/generate
{
  "input": "一个程序员在咖啡馆...",
  "prompt_suffix": "请确保对白简洁...",
  "params": {
    "genre": "都市爱情",
    "duration": 180
  }
}

# 生成角色形象（带提示词覆盖）
POST /api/v1/characters/{id}/generate-image
{
  "prompt_suffix": "请让角色看起来更加阳光...",
  "styles": ["realistic", "anime"],
  "count_per_style": 4
}
```
