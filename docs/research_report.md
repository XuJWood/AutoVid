## 一、市场背景

### 1.1 AI短剧行业现状

AI短剧正在颠覆传统影视制作流程，核心变化：
- **成本降85%**：真人剧单集2万元 → AI制作仅需3000元
- **效率提升百倍**：传统90天 → AI流程10天
- **团队规模缩减**：六人团队三天一部剧 → 单人单日可生成上百条内容

### 1.2 核心痛点

| 痛点 | 描述 | 影响 |
|-----|------|------|
| **角色一致性** | 不同镜头角色"变脸"、服装变化 | 观众无法入戏 |
| **分镜连贯性** | 场景跳跃、逻辑断层 | 作品像"拼贴" |
| **风格统一** | 画面质感忽高忽低、风格混乱 | 品质感缺失 |
| **技术门槛** | 需要多个工具配合、流程复杂 | 制作效率低 |

#### 1️⃣ Huobao Drama（最相关竞品）

**GitHub**: https://github.com/chatfire-AI/huobao-drama

**Stars**: 7,300+（开源不到一个月）

**核心流程**：
```
用户输入主题/文字
      ↓
AI生成剧本（含角色设定、分镜描述）
      ↓
生成角色形象（多套供选择）
      ↓
生成场景图（古风/现代/科幻等）
      ↓
分镜图生成（每个镜头的画面）
      ↓
图生视频（静态画面→动态视频）
      ↓
剪辑合成（配音+字幕+BGM）
      ↓
输出完整短剧
```

**核心功能**：

| 模块 | 功能 | 说明 |
|-----|------|------|
| 剧本解析 | AI解析用户输入，生成结构化剧本 | 含角色、场景、对白 |
| 角色生成 | 多模态模型生成人物形象 | 多套风格可选 |
| 分镜制作 | 剧本自动拆分为分镜 | 场景描述+镜头设计 |
| 图生视频 | 静态分镜图→动态视频 | 支持动作描述 |
| 全局风格 | 锁定整剧视觉风格 | 水墨风/胶片感等 |

**优点**：
- 全流程自动化
- 开源可自部署
- 支持风格锁定
- 角色生成多套可选

**缺点**：
- 生成成本较高（约5-8元/分钟）
- 角色一致性依赖用户手动选择
- 不支持多平台发布
- 技术栈为Go，生态较小

#### 2️⃣ SkyReels（昆仑万维）

**开源地址**: https://github.com/SkyworkAI/SkyReels-V1

**核心能力**：

| 能力 | 说明 |
|-----|------|
| 视频生成 | 一次生成60秒，1080P |
| 角色一致性 | 九宫格参考，保持角色统一 |

**角色一致性解决方案**：

```
└─────────────────┬───────────────────────┘

│         多模态融合编码器                  │

```

## 三、技术调研

### 3.1 角色一致性解决方案

**方案一：Embedding锁定**
- 生成角色后提取特征向量
- 后续生成时注入相同Embedding
- 代表项目：魔因漫创

**方案二：Character ID绑定**
- 上传角色视频/图片获取ID
- 生成时通过`@角色ID`引用
- 代表项目：Sora2 API

**方案三：九宫格参考**
- 提供9张角色参考图
- 模型学习角色特征
- 代表项目：SkyReels-V4

**方案四：3D角色融合**
- 生成3D角色模型
- 渲染时使用同一模型
- 代表项目：SkyReels-A1

### 3.2 分镜生成技术

**流程**：
```
剧本文本 → LLM解析 → 结构化分镜 → 文生图 → 图生视频
```

**分镜数据结构**：
```json
{
  "scene_id": 1,
  "scene_name": "咖啡馆初遇",
  "duration": 5,
  "characters": [
    {
      "name": "李明",
      "character_id": "char_001",
      "action": "推门进入",
      "emotion": "期待"
    }
  ],
  "setting": "现代咖啡馆，午后阳光",
  "camera": "中景，跟随镜头",
  "dialogue": "你好，请问这里有人吗？",
  "visual_prompt": "年轻男子推开门，阳光洒入，咖啡馆内景"
}
```

### 3.3 人物形象生成

**技术路径**：
| 方案 | 工具 | 特点 |
|-----|------|------|
| 文生图 | MidJourney / Stable Diffusion | 风格丰富 |
| 角色一致性 | LoRA / IP-Adapter | 特征固定 |
| 三视图 | 专门训练模型 | 多角度统一 |
| AI数字人 | HeyGen / D-ID | 真人风格 |

## 五、产品差异化机会

### 5.1 现有产品不足

| 问题 | 说明 |
|-----|------|
| 功能割裂 | 生成工具≠发布工具，需多工具配合 |

### 5.2 AutoVid差异化定位

| 维度 | Huobao Drama | AutoVid |
|-----|-------------|---------|
| 目标用户 | 普通创作者 | 创作者+供应商 |

## 七、结论与建议

### 7.1 产品定位确认

**AutoVid = AI短剧创作平台**

核心用户：AI视频创作者/供应商

核心价值：
1. **全流程自动化**：文字→剧本→角色→分镜→视频→发布
2. **角色一致性保障**：Character ID + 多模态参考
3. **批量生产能力**：任务队列 + 模板复用
4. **多平台发布**：一键分发主流短视频平台

### 7.2 MVP功能优先级

| 优先级 | 功能 | 依赖 |
|--------|------|-----|
| P0 | 文字→剧本生成 | LLM API |
| P0 | 角色形象生成（多套可选） | 多模态模型 |
| P0 | 分镜自动生成 | LLM + CV |
| P0 | 图生视频 | 视频生成API |
| P1 | 角色库管理 | 数据库 |
| P1 | 角色衣橱系统 | LoRA / 多图参考 |
| P1 | 抖音发布 | Playwright |
| P2 | 批量任务队列 | Celery |
| P2 | 小红书/B站发布 | Playwright |
